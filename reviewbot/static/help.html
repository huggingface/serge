Getting started with Serge

Serge reviews a pull request with an LLM and posts inline comments on the diff. In the web app you watch the draft stream live and edit it before publishing. Three steps to be up and running:

  1. 1. Install the GitHub App

    Serge posts reviews under a GitHub App identity, so the App needs to be installed on every repo you want to review. Once installed, grant it access to the specific repos you care about — you don't have to give it the whole org.

    Required permissions are Pull requests: R/W, Contents: R, Issues: R, and Metadata: R. The install screen shows these for confirmation.

  2. 2. Add a provider config

    Each review needs an LLM API key. Keys are stored per-repo (or per org) in the app's database so you can route different repos to different providers. Open the Admin page and click Add a provider config:

    • Provider — HF, OpenAI, Anthropic, or a custom OpenAI-compatible endpoint.
    • Default model — used when you leave the model field empty on the submit form.
    • API key — write-only; you can replace it later but never read it back.
    • Repository — exact owner/repo, or an org wildcard like huggingface/*.
    • Allowed users / orgs — GitHub logins or org slugs allowed to spend this key.

    When a review is submitted, the most-specific matching config wins. Submissions without a matching config are rejected.

  3. 3. Start a review

    Head back to New review and fill the form:

    • Pull request — paste a URL or the short form owner/repo#123.
    • Trigger comment — what you'd normally type as a PR comment. Mention @askserge plus any extra instructions (e.g. "focus on the new auth flow").
    • Provider / Model — pre-filled from your admin config; override per-job if you want.
    • Max input tokens — per-review cap on the cumulative input tokens spent across all diff chunks and tool turns. Blank uses the deployment default; 0 disables the cap. Raising it lets the reviewer handle very large PRs, but the cost and latency climb fast.
    • The Presets row (First-pass, New model, Bug fix, Documentation) drops a starter prompt into the trigger comment.

    Hit Start review. The LLM streams its draft into the console; when it finishes you can edit the summary, tweak or discard individual comments, then publish to GitHub. Discarding a draft deletes it — drafts you keep show up under Your reviews for the next 25 jobs.

Tips

  • Drop a .ai/review-rules.md file in the repo's default branch to inject repo-specific instructions into every Serge run.
  • Mentions only trigger a review when the comment author is a MEMBER, OWNER, or COLLABORATOR on the repo — public drive-by comments are ignored.
  • The Journal records every call (who, what PR, which model, token cost) for auditing and billing.
